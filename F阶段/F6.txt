`RV32I` 在手册目录里的位置是第 **2 章**：`2. RV32I Base Integer Instruction Set, Version 2.1`，从 **第 21 页**开始。相关问题主要分布在：
`2.1` 程序员模型、`2.2` 基本指令格式、`2.3` 立即数字段变体、`2.4.2` 整数寄存器-寄存器运算，以及第 `3` 章里的 `RV32E`。

按题目逐个答：

1. `PC` 寄存器的位宽是多少？
`pc` 的位宽是 `XLEN`。对 `RV32I` 来说，`XLEN = 32`，所以 `pc` 是 **32 位**。

2. `GPR` 共有多少个？每个 `GPR` 的位宽是多少？
`RV32I` 有 **32 个通用寄存器**，也就是 `x0 ~ x31`。  
每个 `GPR` 都是 **32 位**宽。

3. `R[0]` 和 `sISA` 的 `R[0]` 有什么不同？
`RISC-V` 里的 `x0`（也就是 `R[0]`）是 **硬连线为 0** 的寄存器：
- 读出来永远是 `0`
- 写进去的值会被忽略

至于你说的 `sISA`，这个不是 RISC-V 手册里的内容，所以这里我只能按课程里常见的 `sISA` 约定理解：
`sISA` 的 `R[0]` 通常是 **普通可写寄存器**，不一定恒为 0。  
所以核心区别就是：**RISC-V 的 `x0` 是常数零寄存器，sISA 的 `R[0]` 一般不是。**
如果你们课程里的 `sISA` 另有特殊定义，那就以课程材料为准，别把两本“教义”混成一本。

4. 指令编码的位宽是多少？指令有多少种基本格式？
`RV32I` 的基础指令都是 **固定 32 位**。

关于“基本格式”这个说法，手册里分两层：
- `2.2` 先说有 **4 种核心格式**：`R / I / S / U`
- `2.3` 又补充了 **2 种变体格式**：`B / J`

所以如果严格按 `2.2` 原话，答案是 **4 种核心格式**；  
如果把变体也算上，常见总数就是 **6 种格式：R / I / S / U / B / J**。

5. 在指令的基本格式中，需要多少位来表示一个 `GPR`？为什么？
需要 **5 位**。

原因很直接：
- `RV32I` 有 `32` 个通用寄存器
- 编码 `32` 个不同寄存器编号至少需要 `log2(32) = 5` 位

所以 `rd`、`rs1`、`rs2` 这些寄存器字段都是 **5 位**。

6. `add` 指令的格式具体是什么？
`add` 是 **R-type** 指令，汇编格式是：

```asm
add rd, rs1, rs2
```

它的编码格式是：

```text
funct7   rs2    rs1   funct3   rd    opcode
0000000  rs2    rs1    000      rd   0110011
```

也就是说：
- 类型：`R-type`
- `opcode = 0110011`
- `funct3 = 000`
- `funct7 = 0000000`

功能就是：`rd = rs1 + rs2`  
溢出直接丢掉高位，只保留低 `XLEN` 位。

7. 还有一种基础指令集称为 `RV32E`，它和 `RV32I` 有什么不同？
`RV32E` 可以看成是 `RV32I` 的“瘦身版”，主要区别是：

- `RV32I` 有 **32 个 GPR**：`x0 ~ x31`
- `RV32E` 只有 **16 个 GPR**：`x0 ~ x15`

其他关键点：
- `RV32E` 仍然是 **32 位**架构
- 指令编码总体上和 `RV32I` 相同
- 只是凡是编码里用到 `x16 ~ x31` 的那些情况，在 `RV32E` 中都 **保留/不可用**

它的设计目标是面向更小的嵌入式核，省面积、省功耗。
